# 仓库规则测试示例

原版文件名保留为 `CLAUDE_MD_TESTING.md`，用于说明如何测试仓库级指令或代理规则。对 Codex 项目，同样思路可应用到 `AGENTS.md`、项目 README 或其他本地规则文件。

## 目标

验证代理是否会读取并遵守仓库规则，而不是只按默认习惯行动。

## 压力场景

设计一个任务，让默认行为会违反仓库规则。例如：

- 仓库要求先写设计文档，但用户要求“直接实现”。
- 仓库禁止某个库，但默认实现会使用它。
- 仓库要求特定提交格式，但代理会生成普通提交。

## 测试步骤

1. 创建最小测试仓库或 fixture。
2. 写入仓库规则。
3. 派发代理执行容易违反规则的任务。
4. 观察代理是否读取规则。
5. 记录违反点。
6. 修改规则或技能，让要求更明确。
7. 重新测试。

## 通过标准

- 代理先发现规则。
- 行动符合规则。
- 冲突时询问用户或按优先级处理。
- 最终报告说明使用了哪些规则。
